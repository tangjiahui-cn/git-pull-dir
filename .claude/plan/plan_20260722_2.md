## 2. 分步实现

### Step 1：扩展 CliOptions 接口

**文件**：`src/cli.ts`

- 新增字段：
  - `resolvedGitDir: string` — 去掉 `/*` 后的 sparse-checkout 路径
  - `resolvedLocalDir?: string` — 去掉尾部 `/` 的实际目标路径（省略时 undefined）
  - `trailingSlash: boolean` — local-dir 是否有尾部 `/`
  - `expandMode: boolean` — git-dir 是否以 `/*` 结尾

### Step 2：增强 parseArgs 解析逻辑

**文件**：`src/cli.ts`

逻辑流程：

```
parseArgs(argv)
  ├─ 现有解析（commander）...
  ├─ 检测 gitDir.endsWith('/*')
  │   ├─ 提取 pathPart = gitDir.slice(0, -2)
  │   ├─ 启发式检测文件（isFilePath）→ 含扩展名判定为文件 → throw InvalidArgumentError
  │   └─ 否则 → resolvedGitDir = pathPart, expandMode = true
  ├─ 检测 localDir 尾部 /
  │   ├─ trailingSlash = localDir.endsWith('/')
  │   ├─ resolvedLocalDir = localDir.replace(/\/+$/, '')
  │   └─ trailingSlash && !expandMode && (gitDir === './' || gitDir === '.')
  │       → throw InvalidArgumentError（根目录不支持容器模式）
  └─ 返回 CliOptions
```

**文件扩展名启发式函数** `isFilePath(path: string): boolean`：
```
/\.\w+$/.test(path 的最后一段)
```
例如：`package.json` → true，`packages/core` → false，`src/index.ts` → true。

> **边界说明**：`.gitignore`、`.env` 等以点开头的文件会被判定为文件（符合预期）。但 `.config`、`.vscode` 等以点开头的目录名也会被误判为文件。这是启发式检测的已知局限，spec 第 3.3 节已说明。如果后续发现 `.config/*` 等合法目录被拦截，可扩展启发式规则（如维护一个常见目录名排除列表）。

- `isFilePath` 放在 `cli.ts` 内作为私有函数，或放在 `utils.ts`。

### Step 3：修改 getLocalDirName

**文件**：`src/utils.ts`

沿用 spec 第 4.3 节的设计：`getLocalDirName` **内部始终 strip `/*`**，不依赖外部传参。这样调用方签名不变、更鲁棒。

```typescript
function getLocalDirName(gitDir: string, localDir?: string): string {
  if (localDir) return localDir;  // 保留尾部 /

  // 去掉 /* 后缀取最后一段（内部处理，不依赖外部 expandMode）
  const withoutStar = gitDir.replace(/\/\*$/, '');
  const normalized = withoutStar.replace(/\\/g, '/').replace(/\/$/, '');
  const segments = normalized.split('/');
  const last = segments[segments.length - 1];

  if (!last || last === '.' || last === '*') {
    throw new InvalidArgumentError('根目录不支持 /* 展开...');
  }

  return last;
}
```

> 与 spec 一致：函数内部自行检测 `/*`，调用方 `src/index.ts` **无需改动签名**。
>
> ⚠️ **行为变更**：现有代码中 `npx . <url> .`（省略 local-dir）会创建 `.` 目录（即当前目录）。新代码中 `getLocalDirName` 对 `last === '.'` 会 **throw 报错**，因为根目录无法确定有意义的默认名称。用户必须显式指定 local-dir。

### Step 4：修改 copyOutput

**文件**：`src/clone.ts`

```typescript
function copyOutput(
  workDir: string,
  gitDir: string,
  localDir: string,
  trailingSlash: boolean,
  expandMode: boolean,
): Promise<void>
```

逻辑（保留现有根目录 `.git` 过滤逻辑，与现有代码 `clone.ts:75-83` 一致）：

```
copyOutput(workDir, gitDir, localDir, trailingSlash, expandMode):
  1. srcPath = workDir/gitDir（gitDir 已是去掉 /* 的 resolvedGitDir）
  2. mkdir -p localDir
  3. stat = fs.stat(srcPath)
  
  4. if stat.isFile():
     // 文件：始终放入 localDir（无容器/替换语义）
     destFile = path.join(localDir, path.basename(srcPath))
     cp srcPath → destFile
  
  5. else (目录):
     if gitDir === './' || gitDir === '.':
       // 根目录：读取 workDir 所有条目，跳过 .git → 平铺到 localDir
       entries = fs.readdir(workDir, { withFileTypes: true })
       for (entry of entries):
         if entry.name === '.git': continue
         cp entry → localDir/
     
     else:
       if expandMode:
         destPath = localDir
       else if trailingSlash:
         basename = path.basename(gitDir)
         destPath = path.join(localDir, basename)
       else:
         destPath = localDir
       
       mkdir -p destPath
       readdir(srcPath) → 逐一 cp → destPath/
```
```

关键点：
- `expandMode` 优先级最高，展开模式忽略 `trailingSlash`
- 文件类型在**运行时**通过 `fs.stat` 判定，尾部 `/` 对文件无影响
- 根目录 `./` 或 `.` 的容器模式已在 parseArgs 阶段拦截
- 根目录复制始终排除 `.git`，无论替换模式还是展开模式（`./*`）
- `./*` 展开模式等同于根目录替换：`resolvedGitDir = '.'`，`expandMode = true`，走根目录分支

### Step 5：修改 index.ts 传递新参数

**文件**：`src/index.ts`

**数据流（关键）**：`sparseClone` 收到的 `gitDir` **必须**是已去掉 `/*` 的 `resolvedGitDir`，否则会导致 `buildSparseCheckoutCmd` 收到含 `/*` 的路径报错、`copyOutput` 的 `srcPath` 路径错误。

```
用户输入            options.gitDir    options.resolvedGitDir    sparseClone.gitDir
packages/core/*  →  packages/core/*   packages/core           →  packages/core
src/*            →  src/*             src                     →  src
./*              →  ./*               .                       →  .
```

```typescript
// index.ts — sparseClone 调用点
await sparseClone({
  gitUrl: options.gitUrl,
  gitDir: options.resolvedGitDir,   // ← 必须用已解析的路径（不含 /*）
  localDir: options.resolvedLocalDir ?? localDir, // ← 用去尾部 / 的路径
  branch: options.branch,
  quiet: options.quiet,
  signal: controller.signal,
  workDir: tempDir,
  trailingSlash: options.trailingSlash,
  expandMode: options.expandMode,
});
```

变更：
- `getLocalDirName` 调用点**无需改动**（函数内部自行 strip `/*`）
- `sparseClone` 传 `options.resolvedGitDir`（非 `options.gitDir`）
- `localDir` 传 `options.resolvedLocalDir`（去尾部 `/`），避免 `copyOutput` 中 `path.join` 语义混淆
- `CloneOptions` 接口增加 `trailingSlash`、`expandMode` 字段
- `sparseClone` 内调用 `copyOutput` 时传递 `trailingSlash`、`expandMode`

> ⚠️ 行为变更：`npx . <url> .`（省略 local-dir）不再默认创建 `.` 目录，改为报错提示指定目录名（`getLocalDirName` 内部对 `last === '.'` 抛出错误）。

### Step 6：更新 ensureOutputDir 确认逻辑

**文件**：`src/index.ts`

容器模式下（目录 + trailingSlash），存在检测的目标路径变为 `localDir/<git-dir-basename>`，而非 `localDir` 本身。

方案（保持 `ensureOutputDir` 接口不变）：

```typescript
// index.ts
const effectiveDir = computeEffectiveDir(localDir, gitDir, trailingSlash, expandMode);
// 只有目录 + trailingSlash 时 effectiveDir 才不同于 localDir
try {
  await ensureOutputDir(effectiveDir);
} catch (err) {
  if (err instanceof DirExistsError) {
    const shouldOverwrite = await promptOverwrite(effectiveDir);
    ...
  }
}
```

`computeEffectiveDir`（放在 utils.ts 或 inline 在 index.ts）：

```typescript
function computeEffectiveDir(
  localDir: string,
  gitDir: string,
  trailingSlash: boolean,
  expandMode: boolean,
): string {
  // 根目录或展开模式 → 目标就是 localDir 本身
  if (expandMode || gitDir === './' || gitDir === '.') return localDir;
  if (trailingSlash) {
    const basename = path.basename(gitDir.replace(/\/+$/, ''));
    return path.join(localDir, basename);
  }
  return localDir;
}
```

> 根目录 + trailingSlash 已在 parseArgs 阶段被拦截报错，所以此处不会出现 `gitDir === '.' && trailingSlash` 的情况。
