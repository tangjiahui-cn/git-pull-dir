# git-pull-dir — 功能规格说明书 v2：通配符与复制语义扩展

## 1. 概述

本规格说明书在 v1（`spec_20260722.md`）基础上，扩展三个新需求：

1. **`/` 尾部斜杠语义** — `[local-dir]/` 表示"复制到该目录"（容器模式），区别于无 `/` 的"替换/创建该目录"
2. **`/*` 通配符语义** — `<git-dir>/*` 表示展开目录内容，复制其下所有条目
3. **`<file>/*` 格式校验** — 文件路径不允许使用 `/*`，报错并给出示例

### 1.1 核心规则：文件 vs 目录的行为差异

尾部 `/`（容器模式）**仅对目录生效**，对文件不产生额外行为：

| git-dir 类型 | 规则 |
|---|---|
| **文件**（如 `src/index.ts`、`package.json`） | 始终复制到 `[local-dir]` 内。尾部 `/` 仅起到"声明目标是目录"的作用，不影响结果。 |
| **目录**（如 `packages/core`、`src`） | 尾部 `/` → 容器模式（在目标下创建子目录）；无 `/` → 替换模式（内容直接放入目标）。 |

> 这是因为：对于文件，"替换目标"无意义（你不可能用文件"替换"一个目录），永远是"放入目录"。

---

## 2. 新命令格式

```bash
git-pull-dir <git-url> <git-dir> [local-dir] [--branch=<name>] [--quiet]
```

### 2.1 参数变更

| 参数 | 变更 | 说明 |
|------|------|------|
| `<git-dir>` | 新增 `/*` 后缀支持 | `packages/core/*` 表示展开目录所有内容 |
| `[local-dir]` | 新增尾部 `/` 语义 | `./output/` 表示将内容复制到该目录下（容器模式） |

## 3. 详细语义

### 3.1 local-dir 尾部 `/` 语义

**规则**：若 `[local-dir]` 以路径分隔符结尾（`/`，Windows 上也为 `/`），则视为"容器模式"——将 git-dir 的内容放置在该目录**下**，而不是让该目录**成为** git-dir 的内容。

| 模式 | 写法 | 行为 | 示例 |
|------|------|------|------|
| **替换/创建模式** (默认，无 `/`) | `npx . <url> packages/core ./output` | `./output` 存放 packages/core 里的文件 | `./output/package.json`, `./output/src/...` |
| **容器模式** (尾部 `/`) | `npx . <url> packages/core ./output/` | 在 `./output/` 下创建 `core/` 子目录，再把 packages/core 内容放进去 | `./output/core/package.json` |

**子目录名称来源**：取 `<git-dir>` 路径的最后一段（`packages/core` → `core`）。此规则与省略 `[local-dir]` 时默认目录名逻辑一致。

**行为对比表**（`@` 表示 git-dir 的 basename）：

| 命令 | git-dir 类型 | 结果结构 |
|------|-------------|----------|
| `... packages/core ./output` | 目录 | `./output/[packages/core 内文件]` — 替换模式 |
| `... packages/core ./output/` | 目录 | `./output/core/[packages/core 内文件]` — 容器模式 |
| `... src/index.ts ./output` | 文件 | `./output/index.ts` — 文件始终放入目标目录 |
| `... src/index.ts ./output/` | 文件 | `./output/index.ts` — 同左，`/` 无额外效果 |
| `... ./ ./output` | 根 | `./output/[根目录内文件]` |
| `... ./ ./output/` | 根 | 报错（根目录无 basename，无法确定子目录名） |

> **根目录 `./` 或 `.` 的容器模式**：git-dir 为 `./` 或 `.` 时，最后一段无法获取有意义的名称。此时**报错提示**：根目录不支持容器模式，请去掉尾部 `/` 或指定明确的 git-dir。因为根目录无"最后一段"概念，无法确定子目录名称。

### 3.2 git-dir `/*` 通配符语义

**规则**：若 `<git-dir>` 以 `/*` 结尾，表示将指定目录下的**所有条目**展开，直接复制到目标位置，而不是将目录本身作为容器。

**本质**：`/*` 等价于"取目录内的所有内容，平铺到目标"。

| 命令 | sparse-checkout 路径 | 复制行为 |
|------|---------------------|----------|
| `... packages/core ./output` | `packages/core` | 将 packages/core 内容复制到 `./output` |
| `... packages/core/* ./output` | `packages/core` | **同左**：将 packages/core 内容复制到 `./output` |
| `... packages/core ./output/` | `packages/core` | 在 `./output/core/` 下放内容 |
| `... packages/core/* ./output/` | `packages/core` | **展开**：将 packages/core 内容直接复制到 `./output/` 下（**不**创建 `core/` 子目录） |

**关键区分**：

| git-dir | local-dir | 结果行为 | 本质 |
|---------|-----------|----------|------|
| `packages/core` | `./output` | `./output` ← packages/core 内容 | 替换/创建 |
| `packages/core` | `./output/` | `./output/core/` ← packages/core 内容 | 容器 |
| `packages/core/*` | (省略) | `./core/` ← packages/core 内容 | 展开，默认名 `core` |
| `packages/core/*` | `./output` | `./output` ← packages/core 内容 | 展开 + 替换 |
| `packages/core/*` | `./output/` | `./output/` ← packages/core 内容 | 展开 + 容器（等价于替换，因为已展开） |
| `./*` 或 `.*` | (任意) | 根目录展开：将仓库根目录内容全部复制到目标 | 根展开 |

#### 3.2.1 省略 [local-dir] 时的默认目录名

当省略 `[local-dir]` 且 git-dir 以 `/*` 结尾时，取 `/*` 之前路径的最后一段作为默认目录名：

| git-dir | 默认 local-dir 名 |
|---------|------------------|
| `packages/core/*` | `core` |
| `src/*` | `src` |
| `./*` 或 `.*` | 报错（根目录无法确定名称） |

#### 3.2.2 `/*` 与容器模式的交互

| 命令 | 结果 |
|------|------|
| `... packages/core/*` | → `./core/` 存放 packages/core 内所有内容 |
| `... packages/core/* ./output` | → `./output` 存放 packages/core 内所有内容（替换模式） |
| `... packages/core/* ./output/` | → `./output/` 内直接存放 packages/core 内容（容器模式，但因为没有包裹层，效果同替换） |

> `/*` 本质已是"展开"，容器模式在此场景下意义较小，但语法上同时支持两种标记时，`/*` 优先级更高（展开语义覆盖容器语义）。即：`packages/core/* ./output/` → 展开到 output/，不在 output/ 下创建子目录。

### 3.3 `<file>/*` 格式校验

**规则**：解析 `<git-dir>` 时，若其以 `/*` 结尾，检查 `/*` 之前的路径是否指向一个**文件**（而非目录）。若判定为文件，则报错。

**判断方法**：在参数解析阶段做启发式检测：
- 若路径最后一段包含文件扩展名（`.` 后跟字母数字，如 `.json`, `.ts`, `.js`, `.md`），判定为文件
- 若路径最后一段无扩展名（如 `packages/core`），判定为目录

**错误信息**：
```
Error: "package.json/*" 格式错误 —— "package.json" 是一个文件，不能使用 /* 进行展开。
请使用目录路径，例如：
  npx . <url> src/*
  npx . <url> packages/core/*
  npx . <url> ./*
```

**输出要求**：
- 退出码：`1`
- 错误信息明确指出具体路径、错误原因，并给出至少 2 个正确示例

## 4. 内部实现变更

### 4.1 CliOptions 接口扩展

```typescript
interface CliOptions {
  gitUrl: string;
  gitDir: string;           // 原始 git-dir 值，如 "packages/core/*"
  resolvedGitDir: string;   // 解析后的 sparse-checkout 路径，如 "packages/core"（去掉 /*）
  localDir?: string;        // 原始 local-dir 值
  resolvedLocalDir: string;  // 解析后的实际目标路径（去掉尾部 /）
  trailingSlash: boolean;   // local-dir 是否有尾部 /
  expandMode: boolean;      // git-dir 是否以 /* 结尾（展开模式）
  branch: string;
  quiet: boolean;
}
```

> **设计说明**：`trailingSlash` 仅记录用户输入的标记，不做为最终行为决策。`copyOutput` 在**运行时**通过 `fs.stat` 判断路径类型，再决定容器语义是否应用。这是为了准确处理"只有目录才受尾部 `/` 影响"的规则——解析阶段无法获知远程路径是文件还是目录。

### 4.2 copyOutput 逻辑变更

当前 `copyOutput` 对目录的处理是：`readdir(srcPath)` 后逐个复制条目到 `localDir`。这已经是"展开"行为。

变更点：

```
copyOutput(workDir, gitDir, localDir, trailingSlash):
  1. 确定 srcPath = workDir/gitDir（去掉 /*）
  2. 创建 localDir（mkdir -p）
  3. 检查 srcPath 是文件还是目录:
     a) 文件:
        - destFile = path.join(localDir, basename(srcPath))
        - cp srcPath → destFile
        - 尾部 / 对文件无影响
     b) 目录:
        - 若 trailingSlash: destPath = localDir/<git-dir最后一段>
        - 否则: destPath = localDir
        - 创建 destPath（mkdir -p）
        - readdir(srcPath) → 逐条目 cp → destPath/
```

> **核心变化**：copyOutput 现在接受 `trailingSlash` 布尔值（而非 `copyMode`），在**运行时**通过 `fs.stat` 判断 git-dir 是文件还是目录，再决定是否应用容器语义。这是为了准确处理"只有目录才受尾部 `/` 影响"的规则——解析阶段无法获知远程路径是文件还是目录。

### 4.3 getLocalDirName 变更

需处理 `/*` 后缀：

```typescript
function getLocalDirName(gitDir: string, localDir?: string): string {
  if (localDir) {
    return localDir;  // 保留尾部 /，由调用方判断容器模式
  }
  
  // 去掉 /* 后缀取最后一段
  const withoutStar = gitDir.replace(/\/\*$/, '');
  const normalized = withoutStar.replace(/\\/g, '/').replace(/\/$/, '');
  const segments = normalized.split('/');
  const last = segments[segments.length - 1];
  
  if (!last || last === '.') {
    throw new InvalidArgumentError(
      '根目录不支持 /* 展开，请在 <git-dir> 中指定具体目录路径。示例：src/* 或 packages/core/*'
    );
  }
  
  return last;
}
```

### 4.4 parseArgs 变更

```typescript
function parseArgs(argv: string[]): CliOptions {
  // ... 现有解析逻辑 ...
  
  let expandMode = false;
  let resolvedGitDir = gitDir;
  let trailingSlash = false;
  
  // 新增：检测 /* 并校验
  if (gitDir.endsWith('/*')) {
    const pathPart = gitDir.slice(0, -2);
    if (isFilePath(pathPart)) {  // 启发式检测文件扩展名
      throw new InvalidArgumentError(
        `"${gitDir}" 格式错误 —— "${pathPart}" 是一个文件，不能使用 /* 进行展开。\n` +
        `请使用目录路径，例如：\n` +
        `  npx . <url> src/*\n` +
        `  npx . <url> packages/core/*`
      );
    }
    resolvedGitDir = pathPart;
    expandMode = true;
  }
  
  // 新增：检测 local-dir 尾部 /
  if (localDir) {
    trailingSlash = localDir.endsWith('/');
    // 仅对目录且非 expand 模式检查根目录容器 → 报错
    if (trailingSlash && !expandMode && (gitDir === './' || gitDir === '.')) {
      throw new InvalidArgumentError(
        '根目录 (./ 或 .) 不支持容器模式（尾部 /），因为无法确定子目录名称。\n' +
        '请去掉 local-dir 尾部 / 或指定明确的 git-dir。'
      );
    }
  }
  
  return {
    gitUrl,
    gitDir,
    resolvedGitDir,
    localDir,
    resolvedLocalDir: localDir ? localDir.replace(/\/+$/, '') : undefined,
    trailingSlash,
    expandMode,
    branch: options.branch || 'main',
    quiet: !!options.quiet,
  };
}
```

> **注意**：`expandMode` 与 `trailingSlash` 可以同时为 true。在 copyOutput 中处理优先级：`expandMode` 优先（展开模式始终平铺内容到目标，不需要子目录包裹）。

### 4.5 稀疏检出路径

`/*` 模式不影响 sparse-checkout 的路径参数：

| 用户输入 | sparse-checkout set |
|----------|-------------------|
| `packages/core` | `packages/core` |
| `packages/core/*` | `packages/core`（去掉 `/*`） |
| `./*` | `./`（根目录，触发跳过 sparse-checkout） |

## 7. 边界情况与注意事项

### 7.1 `/*` 与文件系统安全

- `/*` 在 shell 中是 glob 展开符号，但在本工具参数中作为字面字符串处理，不会触发 shell 展开（参数未经过 shell glob）
- 用户应当传递字面字符串 `src/*`，而非让 shell 展开成 `src/file1 src/file2` 等

### 7.2 容器模式路径重叠

- `npx . <url> packages/core ./output/` 创建 `./output/core/`
- 若 `./output/core/` 已存在，触发覆盖确认逻辑（同现有 `DirExistsError` 处理）

### 7.3 git-dir 尾部 `/` 的兼容性

当前 `getLocalDirName` 中有 `replace(/\/$/, '')` 去掉 git-dir 尾部的 `/`。`/*` 不受影响，因为 `/*` 不是 `/$`。但若有用户输入 `packages/core/`（尾部单斜杠），去掉尾部 `/` 后等价于 `packages/core`。

### 7.4 Windows 兼容性

- 尾部 `/` 在 Windows 上也使用 `/` 表示（Node.js path 模块统一处理）
- `/*` 在 Windows 上也使用 `/*` 格式
